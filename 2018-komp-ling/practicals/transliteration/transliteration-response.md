game -- текстовый файл, на материале которого проводилось тестирование работы программ

freq -- частотный список слов текста game
freq_sorted -- отсортированный частотный список слов текста game

alphabet -- словарь для транслитерации
translite_text -- транслитерированный текст

Были созданы программы:
tokenizer.py -- токенизация текста с помощью модуля nltk
frequency.py -- создание словаря частотности для текста  
rank.py -- сортировка частотного словаря по частотности слов и запись отсортированного частотного словаря в файл
ranking.py -- присвоение слову позиции в словаре частотности
transliterate.py -- транслитератор текста с кириллицы (русский алфавит) в латиницу (словарь соответствий для транслитерации в файле alphabet)

(более подробно функции программ описаны в комментариях внутри программ)

Вопросы по работе транслитератора:
1. Для разрешения неоднозначности можно установить правила (например, если "е" в начале слова или после гласной, она переходит в "je", а если после согласной, то в "е")
 
2. Можно сделать правила, согласно которым, если две буквы встречаются вместе, то они переходят в одну (при этом может понадобиться список исключений в случае, если правила работают не для всех случаев)

3. Звучание фонемы может зависеть от её позиции в слове. Например, если мы хотим сделать транслитерацию фонетически, а не орфографически точной, то мы можем оглушать русские звонкие согласные на конце слова. Например, "г" в слове "друг" перейдет в "k", а "г" в слове "горб" перейдет в "g".
